## Notes / Tradeoffs
- **Radix AvatarImage + jsdom:** `@radix-ui/react-avatar` conditionally mounts the image based on load status, which is not deterministic in jsdom. `UserAvatar` now uses a native `<img>` (with shadcn-equivalent classes) to keep tests deterministic and still provide fallback-on-error behavior.
- **DetailPanel a11y + testing:** `SheetContent` requires title/description. `DetailPanel` includes sr-only `SheetTitle` and `SheetDescription`. The visible header title means the title string appears twice; tests assert the dialog’s accessible name and use a dedicated close `data-testid`.
- **JSX runtime in tests:** `apps/web/vitest.config.ts` uses esbuild `jsx: "automatic"` to avoid `React is not defined` failures with TSX.
